What makes it different

This is not a cloud service pretending to be private. It is a self-contained encryption tool designed around local processing, portability, and transparency.

⛁

Local-first by design

Text is encrypted and decrypted directly in the browser. The server does not process, store, or inspect user content.

⇄

Portable across devices

Encrypt on one machine, move the file by USB or email, then decrypt it elsewhere with the same password.

⌘

Zero-install simplicity

A single HTML file can be carried offline and used without installing software or creating an account.

⚑

Open source and auditable

Anyone can inspect the code, verify the workflow, and run the tool offline if they prefer.

🔐

Modern encryption

Uses AES-GCM with a password-derived key for practical, current browser-based encryption.

☍

Broad compatibility

Tested successfully across major browsers on Windows, macOS, iPhone, and multiple Linux environments.

How it works

The workflow is intentionally simple so a person can understand what is happening instead of blindly trusting a black box.

1

Enter the text

Type or paste the note, message, or content to protect.

2

Set a password

Create a strong password or passphrase. The tool checks strength and requires a minimum standard.

3

Encrypt and download

The browser encrypts the content locally and saves the result as a portable file, such as private-note.pvault.

4

Share the encrypted file

Send the encrypted file by email, cloud storage, chat, or USB. The file can safely travel without exposing the original text.

Best practice: If you share an encrypted file with someone else, send the password through a different channel. For example, email the file and text the password separately.

Trust, transparency, and privacy

Encryption tools live or die on trust. This one is designed to keep the trust story simple and verifiable.

No accounts. No tracking. No content storage.

The tool is designed so users do not need to create an account or provide personal information. Content is handled locally in the browser rather than sent to a backend for processing.

Open source for peace of mind

The source code can be published for anyone to inspect, download, and review. That makes the privacy promise easier to verify instead of asking users to accept it on faith.

Portable and offline-friendly

A user can carry the HTML file and encrypted notes on a USB drive, then open the tool on another modern browser without installing software.

Important limits

If a password is forgotten, the text cannot be recovered. And if a person decrypts sensitive content on an untrusted or public computer, the surrounding machine may still pose a risk.

Frequently asked questions

Short answers for the questions people are likely to ask first.

Can I email an encrypted file to someone else?

Yes. You can encrypt the text, download the .pvault file, and email it to someone anywhere. They can decrypt it using the same tool and the correct password. For better security, send the password through a different channel.

Does this work under European privacy expectations?

The privacy model is very simple: the tool is designed to process content locally in the browser without collecting personal data or storing user content on a server. That makes the compliance story much cleaner than a typical web service.

Can the password be recovered if it is forgotten?

No. That is part of the privacy promise. The system is designed so the password is not recoverable by the tool operator.

Can I use it on a public computer?

Technically yes, but it is not recommended for sensitive material. A public or shared computer may have monitoring software, browser extensions, or other risks outside the encryption tool itself.
